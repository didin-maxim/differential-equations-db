# Реализация режима «Симуляция экзамена»

Дата: 2026-05-27.

## Что добавлено

- В viewer добавлен быстрый режим `Симуляция экзамена`.
- Студент выбирает начальный уровень от 3 до 10.
- Симуляция задает серию из 4-5 вопросов: после правильного ответа текущий уровень повышается, после ошибки понижается.
- При падении уровня до 2 симуляция завершается провалом.
- На панели показываются текущий уровень, оценочный разброс баллов, число ответов, зачтенных вопросов и покрытых тем.
- Подбор вопросов старается менять фрагменты курса и исключает олимпиадные задачи, `beyond_standard_course`, карточки с незавершенным решением и вычислительно тяжелые задачи.
- Поддержаны типы интерактива:
  - `self_check`: честная самопроверка доказательства или решения;
  - `numeric` / `formula`: короткий проверяемый ответ;
  - `multiple_choice`: выбор метода или ответа.
- Идеи и решения остаются скрытыми до ответа.

## Архитектура

Режим встроен в `tools/build_viewer.py` и работает поверх текущего `index/generated.json`.

Поддержан metadata overlay из `data/exam_simulation`:

- `DB.exam_simulation.question_overlays[]`;
- legacy/fallback: `DB.exam_simulation.cards[]`;
- ключ карточки `card_id` или `id`;
- поля `answer_mode`, `prompt_override`, `auto_check`, `target_score`, `topic`;
- для старого формата также понимаются `type`, `prompt`, `answers`, `choices`, `success`, `failure`.

В текущем индексе подключены `36` overlay-вопросов версии `1`.

Если overlay отсутствует, viewer использует fallback:

- берет кандидатов из существующих карточек;
- классифицирует теоремы/леммы как `self_check`;
- для части компактных задач использует встроенные проверяемые интерактивы;
- для остальных задач с короткой техникой строит multiple-choice по стандартной идее.

## Правила окончания

- До 4 вопросов симуляция не заканчивается, кроме провала.
- После 4 вопросов симуляция завершается, если результат уже достаточно определен: сильный устойчивый результат, явный слабый результат или одинаковый исход двух последних ответов.
- Если результат шумный, задается пятый вопрос.
- После 5 вопросов симуляция завершается всегда.

node -e "... new Function(inlineScript) ..."

Browser-smoke через локальный HTTP viewer:

- кнопка `Симуляция экзамена` есть;
- уровни старта: `3,4,5,6,7,8,9,10`;
- `DB.exam_simulation.exam_simulation_version = 1`;
- `DB.exam_simulation.question_overlays.length = 36`;
- после старта видны текущий уровень и разброс баллов;
- появляется вопрос и элементы ответа;
- `details` с идеями/решением закрыты до ответа.

## Ограничения первой итерации

- Автоматическая проверка полноценно работает только для карточек с overlay или встроенными короткими интерактивами.
- Большинство теоретических и идейных вопросов остаются в режиме самопроверки, что соответствует формату симуляции устного экзамена.
- Баланс типов вопросов эвристический; его стоит усиливать отдельным overlay после аудита конкретных экзаменационных карточек.
