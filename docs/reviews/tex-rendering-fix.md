# Правка отображения TeX в viewer

Дата: 2026-05-27.

## Что было

`tools/build_viewer.py` безопасно вставлял пользовательские поля через HTML-экранирование, но после этого никак не обрабатывал математическую разметку. Поэтому фрагменты TeX/LaTeX из `title`, `statement`, `ideas`, `solutions` и близких видимых полей оставались сырой строкой. Старые карточки часто используют не полный LaTeX, а компактную ASCII-запись вроде `q(x)>=1`, `x -> +infty`, `[a,a+pi]`, что ухудшало читаемость даже без явных `$...$`.

## Что исправлено

- В viewer добавлен KaTeX 0.16.10 через jsDelivr CDN, пригодный для GitHub Pages.
- Подключен `auto-render` с delimiters `$$...$$`, `\\[...\\]`, `\\(...\\)` и `$...$`.
- Рендер запускается после первичной загрузки и после каждого динамического обновления списка карточек.
- Все пользовательские данные по-прежнему сначала проходят через `esc`; KaTeX применяется к уже вставленным текстовым узлам. Настройки: `throwOnError: false`, `strict: 'ignore'`, `trust: false`.
- Если KaTeX не загрузился, включается встроенный fallback: он безопасно разбирает только явно размеченные delimiters и показывает формулы выделенным математическим текстом без интерпретации HTML.
- Для старых неделимитированных ASCII-фрагментов добавлен очень узкий слой читаемости: `->`, `<=`, `>=`, отдельные `pi`, `infty` отображаются как `→`, `≤`, `≥`, `π`, `∞`.
- Маркерная карточка `oral-exam-strong-10-sturm-comparison-one-zero` теперь в карточке отображает `[a,a+π]`, `(a,a+π]`, `q(x)≥1`, а не сырой `[a,a+pi]` / `>=`.

## Ограничения

- Viewer не пытается автоматически распознавать и оборачивать произвольную ASCII-математику в TeX: это было бы рискованно и могло бы портить обычный текст.
- Полноценный красивый рендер работает для формул, явно размеченных delimiters. Для старого plain-text math сделана только безопасная типографская нормализация самых частых символов.
- Если CDN KaTeX недоступен, страница остается рабочей и безопасной; явно размеченные формулы отображаются fallback-слоем, но без полноценной TeX-верстки дробей, матриц и сложных конструкций.
