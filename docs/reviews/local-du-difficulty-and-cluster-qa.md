Практическая рекомендация:

- `idea_score=5` держать для стандартного шаблона курса: линейное первого порядка, жорданова клетка, простая замена, базовый метод характеристик.
- `idea_score=6` ставить для письменного экзамена с одной вариацией стандартного метода: параметрическая производная, свободный конец, подобранная замена, два первых интеграла характеристик.
- `idea_score=7` оставлять, когда есть дополнительный концептуальный слой: вторая вариация со спектральным/знаковым выводом, энергетическая единственность плюс стрельба, неосцилляционный аргумент.
- Длинный интеграл, громоздкая первообразная или неприятная алгебра повышают `technical_score`, но не `idea_score`.
- Если тяжелый интеграл вообще не влияет на решение, как инертное слагаемое `sin(x^2)` в функционале, не повышать ни `idea_score`, ни `technical_score` из-за него.
- Если задача специально обходится коротким инвариантом, энергией или заменой, повышать `idea_score`, а не технику.

## Дальнейшие правила QA

1. При ревью письменных экзаменов сначала сравнивать с устной шкалой: если задача решается как устная `middle_5_7`, не поднимать `idea_score` только из-за письменного происхождения.
2. Для задачников и письменных вариантов отделять "найти замену/инвариант" от "после этого долго считать".
3. В комментарии difficulty желательно писать оба основания явно: `idea_score=...; technical_score=...`.
4. Для кластеров с `status=watch` новая карточка должна закрывать пункт из `allowed_variants`, а не просто быть еще одним вариантом того же решения.
