# Viewer Navigation and TeX Rules Hardening

Дата: 2026-05-27.

Повод: пользовательская проверка выявила несколько P1-проблем, которые прежние
аудиты не должны были пропускать:

- root-страница не воспринималась как внятная главная;
- кнопки задач в кластерном режиме выглядели как действия, но не открывали
  отдельную задачу понятным маршрутом;
- после открытия карточки не было надежного обратного пути к кластеру/поиску;
- method guide мог оказаться в одной выдаче с задачами и восприниматься как
  обычная карточка;
- в видимой части базы оставались сырые TeX/ASCII-фрагменты вроде `int_a^b` и
  `F_{y'}`.

## Новые редакционные правила

Добавлено в `docs/BASE_WORKFLOW_RULES.md`:

- главная страница базы является навигационным входом, а не списком карточек;
- кластерная страница должна отделять методический блок от списка задач;
- кнопка задачи обязана открывать одиночную карточку, а не просто выставлять
  поиск;
- одиночная карточка обязана иметь заметные пути назад к кластеру и поиску;
- положительный UX-аудит недействителен без сценарных кликов по root,
  кластеру, задаче и обратному пути.

Добавлено в `docs/TASK_CLUSTERS.md`:

- обязательные переходы `каталог кластеров -> кластер -> карточка -> кластер`
  и `карточка -> поиск`;
- старые фильтры при переходе к конкретному кластеру должны сбрасываться или
  быть изолированы, чтобы не создавать пустые страницы.

Добавлено в `docs/AI_CARD_RULES.md`:

- в видимом тексте нельзя оставлять сырые математические исходники без
  TeX-разметки или человеческой переписи;
- после исправления одной формулы нужно проверять условие, идеи, решение,
  экзаменационные критерии и relation-тексты рядом.

## Валидатор

`tools/audit_rules.py` получил опциональный режим:

```powershell
python tools\audit_rules.py --check-raw-tex
```

Он ищет вероятный сырой TeX в видимых полях. Режим пока не включен в обычную
precommit-проверку, потому что исторический корпус дает сотни предупреждений.
Для новых и отредактированных карточек предупреждение из этого режима нужно
считать блокирующим.

## Критерий приемки для следующих аудиторов

Фраза "база удобна" запрещена как итог без доказательства. Аудитор должен
указать:

- какой URL открыл;
- какие кнопки нажал;
- какие фильтры были активны;
- сколько задач увидел;
- отделены ли method guide и задачи;
- открылась ли одиночная карточка;
- куда ведут кнопки назад.

Если хотя бы один из этих пунктов не проверен, аудит считается обзором, а не
приемкой.
