# Аудит viewer для слабого студента: быстрый поиск задач на 3-4

Дата: 2026-05-27.

Роль проверки: студенту нужно быстро найти простые задачи на 3-4 и понять, что учить, особенно с телефона.

## Проверенные сценарии

- Простая выборка по сложности: `idea_score <= 4`, `technical_score <= 4`.
- Темы: первый порядок, линейные уравнения, системы.
- Поиск: `интегрирующий множитель`, `Коши`, `Штурм`, `устойчивость`.
- UX: понятность кнопок, видимость активных фильтров, счетчики, узкий экран, длинные названия, скрытость решений по умолчанию.

## Что исправлено

- Фильтр `Диапазон` переименован в `Уровень по баллам`, чтобы выбор `простые: идея/техника <= 4` был понятнее студенту, который не знает внутренние поля базы.
- Активные фильтры теперь дублируются над результатами, а не только в боковой колонке. На мобильном экране студент видит, почему список сузился, не возвращаясь к началу фильтров.
- Усилены переносы длинных чипов, плашек и карточек: добавлены ограничения для длинных названий кластеров, тегов, id и статусов на узком экране.
- На мобильной ширине панель сортировки уходит на отдельную строку, чтобы не спорить за место со счетчиками.

## Результаты по простому порогу

По данным текущего индекса:

- Всего карточек: 357.
- `idea_score <= 4` и `technical_score <= 4`: 62 карточки.
- То же без олимпиадных карточек: 58 карточек.

В простом неолимпиадном слое:

- Первый порядок: 23 карточки.
- Линейные уравнения: 24 карточки.
- Системы: 4 карточки.

Поиск в простом неолимпиадном слое:

- `интегрирующий множитель`: 6 карточек.
- `Коши`: 15 карточек.
- `Штурм`: 0 карточек.
- `устойчивость`: 10 карточек.

## UX-наблюдения

- Основной путь для студента на 3-4 теперь читается так: выбрать `Уровень по баллам -> простые: идея/техника <= 4`, при необходимости включить `без олимпиадных`, затем искать по теме или слову.
- Счетчики есть в select-опциях, быстрых кнопках, фасетах и верхней сводке. После правки активные фильтры видны рядом с результатами, что снижает риск "потеряться" на мобильном.
- Быстрые кнопки учебных режимов в целом понятны, но `Письменный минимум` не является строго тем же самым, что `idea_score <= 4` и `technical_score <= 4`: режим шире и может включать `idea_score`/`technical_score` до 5 или карточки по экзаменационным тегам. Это стоит оставить как содержательное правило, а не молча менять UI.
- Решения скрыты по умолчанию: в сгенерированном HTML не найдено раскрытых `<details open>`.

## Содержательные замечания без правки данных

- Для `Штурм` на пороге `idea_score <= 4`, `technical_score <= 4` нет результатов. Это может быть нормально, если тема заведомо выше уровня 3-4, но студенту полезно явно понимать, что пустой результат вызван порогом сложности.
- В системах на простом неолимпиадном слое мало карточек: текущий путь годится для старта, но выборка тонкая.
- Поиск `Коши` широкий: он находит не только начальные задачи первого порядка, но и краевые/теоретические карточки, где слово встречается в тексте.

## Ограничения проверки

In-app browser backend в этой сессии не был доступен, поэтому мобильная проверка выполнена статически по HTML/CSS и счетчикам индекса, без интерактивного скриншота.
