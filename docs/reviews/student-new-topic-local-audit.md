# Локальный студенческий проход: новая тема и подготовка к контрольной

Дата: 2026-05-27.

Роль: студент, который открывает новую тему и хочет за 30-60 минут понять метод, затем решить несколько задач для самопроверки.

## Что стало хорошо

1. Почти все учебные крупные кластеры теперь имеют методический блок. Для студента это принципиально меняет базу: можно начинать не с случайной задачи, а с короткого навигатора.
2. В карточках появились `definition_ids`, а viewer получил навигацию по определениям. Это помогает идти от понятия к задачам.
3. Внутри крупных кластеров есть распределение по сложности: простые задачи, средний уровень, сильные/олимпиадные представители.
4. У ключевых тем есть нормальная цепочка:
   - линейное ОДУ первого порядка -> интегрирующий множитель -> точные формы;
   - системы -> фундаментальная матрица -> матричная экспонента -> Флоке;
   - устойчивость -> первые интегралы -> энергетические оценки;
   - краевые задачи -> Штурм -> функции Грина -> резонанс;
   - вариационное исчисление -> краевые/спектральные задачи.

## Проверенные маршруты

### Линейное ОДУ первого порядка

Маршрут найден хорошо: `linear-first-order-ode` имеет method guide и простые задачи уровня 3-4 (`linear-cooling-law`, `weak-pass-linear-method-choice`, `resit-pass-3-linear-first-order-forcing`). Дальше есть задачи 5-7 и олимпиадные применения.

Проблема: для студента не выделены первые 2-3 задачи после чтения guide. Нужно UI-выделение “начать здесь”.

### Интегрирующий множитель

Кластер покрыт хорошо: exact equation, `mu(x)`, `mu(y)`, `mu(xy)`, степенной множитель. Методический блок полезен.

Проблема: сложность `cluster-integrating-factor-mu-xy` имеет idea 6 / tech 2, но визуально рядом с простыми задачами; студент может взять его слишком рано. Нужна сортировка/группировка внутри маршрута.

### Системы и матричная экспонента

Маршрут богатый: есть диагональная система на минимум, жорданова клетка, восстановление системы, trace/det shortcut, матричная рекуррента.

Проблема: в representatives встречаются сильные Putnam-задачи с `technical_score` 6-7. Для контрольной они должны быть явно отнесены к “сложнее/олимпиадное”.

### Устойчивость и первые интегралы

Очень хороший слой для визуального обучения: есть простые phase-line/phase-plane карточки с технической сложностью 1-2, затем Ляпунов, первые интегралы, нелинейные системы.

Проблема: кластер `phase-line-stability` очень большой; без подмаршрута студент видит слишком много карточек сразу.

### Краевые задачи и Штурм

Методические блоки дают правильную лестницу: простая задача Дирихле -> собственные значения -> резонанс -> Штурм -> функции Грина. Есть хорошие задачи уровня 3-6.

Проблема: часть штурмовских задач имеет высокий идейный уровень; нужна явная пометка “после базового BVP”.

### Вариационное исчисление

Method guide покрывает почти все стандартные постановки. Есть задачи на свободный конец, трансверсальность, изопериметрию, вторую вариацию.

Проблема: простого входа уровня 3-4 почти нет. Первая реальная задача в representatives начинается примерно с idea 5 / tech 5. Для контрольной нужен “нулевой вход”: выписать Эйлера-Лагранжа и решить короткую задачу без тяжёлого интеграла.

## Главная UX-проблема

Студент может найти кластер и method guide, но после guide ему всё ещё нужно самому понять порядок задач.

Лучшее маленькое улучшение: для method guide во viewer показывать рядом подборку representative-задач того же кластера:

- “Начать”: idea <= 4 или exam_score_3/4, technical <= 4;
- “Контрольная”: idea 5-7, technical <= 6;
- “Сложнее”: idea >= 8 или olympiad tags, либо technical > 6.

Это можно сделать на уровне viewer без изменения данных.

## Содержательные предупреждения

`audit_rules.py` правильно подсвечивает 4 карточки с `exam_score_*` и `technical_score > 5`:

- `local-du-written-2024-variational-free-endpoint`;
- `local-du-written-2014-51-characteristics-pde`;
- `local-du-written-2014-51-lagrange-singular-curves`;
- `local-du-written-2014-51-factorized-variable-coeff`.

Они могут оставаться экзаменационными письменными задачами, но не должны автоматически попадать в устную симуляцию и не должны выглядеть как короткая самопроверка.

## Приоритет

1. Добавить в viewer study-route блок для method guide.
2. Для вариационного исчисления добавить 1-2 очень простые входные карточки или явно показать теоретические mini-steps как первые упражнения.
3. После завершения всех агентов запустить финальный QA по student routes и exam simulation.
