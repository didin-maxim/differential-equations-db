## Проверка кандидатов

Фильтр устной симуляции:

- разрешены `problem`, `theorem`, `lemma`, `definition`;
- разрешены только `standard_course_methods` и `advanced_standard_course`;
- исключены `olympiad_above_exam`, `beyond_standard_course`,
  `needs_solution_completion`;
- для задач требуется `technical_score <= 5`.

Главный дефицит для будущей реализации - строгий уровень `10`: карточек с
`idea_score` в диапазоне `9-10` и технической сложностью до `5` пока только `5`.
Это не блокирует запуск режима, потому что конфиг допускает соседний уровень, но
для хорошей симуляции сильного устного экзамена стоит добавить еще несколько
коротких идейных задач уровня `9-10` в рамках стандартного курса.

## Решения для реализации

- Основная шкала адаптации - `idea_score`; `technical_score` только отсекает
  вычислительных монстров.
- Оценка хранится как вещественный `rating`, студенту показывается диапазон.
- Серия заканчивается за `4` вопроса, если результат уже устойчив; иначе задается
  пятый вопрос.
- Провал наступает сразу при рабочем уровне `2`.
- Баланс вопросов задается типами `proof`, `method`, `application`, `thinking`.
- До появления поля `interactive` viewer может использовать эвристики и
  self-check для теорем/лемм.
