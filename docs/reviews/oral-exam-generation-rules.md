# Правила Генерации Устных Экзаменационных Задач

Цель прохода - добавить типовые задачи устного экзамена МФТИ по дифференциальным
уравнениям. Это не задачи на воспроизведение определения, формулировки или
доказательства стандартной теоремы. Каждая карточка должна проверять применение
знаний из курса.

## Общие требования

- Низкая или умеренная техническая сложность: студент должен думать о методе, а
  не тонуть в вычислениях.
- Задача соответствует стандартной программе ОДУ.
- Формулировка короткая, однозначная, без громоздких параметров.
- Решение полное и самодостаточное.
- Обязательно заполнены `difficulty.idea_score` и `difficulty.technical_score`.
- Должен быть ровно один методический тег из:
  `standard_course_methods`, `advanced_standard_course`, `beyond_standard_course`.
- Для собственно составленных задач используется источник
  `src-mipt-oral-exam-model-tasks` и тег `self_authored`.

## Роли Экзаменатора

Пачки задач разделены по легендам:

- `resit_pass_3`: пересдача; студент просит поставить 3, нужно проверить минимум.
- `weak_pass_3`: слабый студент; нужно понять, знает ли он на 3 или идет на пересдачу.
- `above_three`: обычный экзамен; нужно понять, заслуживает ли студент выше тройки.
- `middle_5_7`: средний студент; выбор между хор(5), хор(6), хор(7).
- `excellent`: проверка знания на отлично.
- `strong_10`: сильный студент претендует на отл(10), задача повышенной сложности, но по теме курса.

## Избегать

- Дубликатов уже существующих карточек.
- Задач вида "сформулируйте теорему" или "докажите стандартную теорему".
- Тяжелых неопределенных интегралов как главной трудности.
- Неясных "найти все" без указания промежутка или условий гладкости.
- Задач, где ответ зависит от внешней теории вне курса, если это не специально
  помечено `beyond_standard_course`.
